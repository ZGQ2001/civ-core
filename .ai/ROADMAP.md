# 产品路线图

> 只看做什么、为什么、怎么做完。不讨论技术实现。

---

## 一、工程特点

土木检测内业和普通软件开发有两个根本不同：

**1. 数据是死的，规范是活的。**
Excel 里 100 根锚杆的位移读数不会自己变，但 GB 50086 和 JGJ 106 的判定公式不同。同一组数据，换个规范就可能从合格变不合格。所以计算和规范必须严格绑定——选了什么规范就出什么结果，不能混。

**2. 报告格式由甲方定，不由我们定。**
每个甲方有自己的 Word 模板。行高列宽、合并单元格、签字栏位置——全都不一样。程序不能假设「表头一定在第 3 行」，它得能适应任何甲方给的模板。

**3. 出错代价不是报个异常，是工程事故。**
一份锚杆报告判错了合格/不合格，楼可能就按错的参数施工。所以每一步必须可追溯——哪组数据、用了哪个规范公式、得出什么判定——能倒回去查。

---

## 二、行业使用中必须规避的风险

| 风险 | 规避方式 |
|------|---------|
| 规范用错 | 每次计算锁定规范版本，报告里注明判定依据 |
| 数据串行（A 项目数据填到 B 项目报告里） | 工作区隔离，一个项目一个文件夹 |
| 报告被误改 | 输出 PDF 归档，docx 只作中间产物 |
| 公式实现错误 | Python 和 C# 双算对比（迁移期），xUnit 覆盖所有判定路径 |
| 用户不懂程序逻辑 | 每个操作有日志反馈，判定结果附公式和中间值 |

---

## 三、与其他工程软件的最大区别

**传统土木软件（PKPM、YJK、理正）**：人操作软件，一个按钮一个功能，所有步骤人来做。

**civ-core**：第一阶段和第二阶段跟传统软件一样——人操作工具。第三阶段开始分叉——人变成决策者，AI Agent 变成操作者。人只说"这批锚杆用 GB 50086 出报告"，Agent 自己找数据、选模版、填报告、转 PDF，人最后看一眼判定结果就完了。

这不是换个 UI 的问题，是操作模式从「人指挥软件」变成「人指挥 AI，AI 指挥软件」。

---

## 四、第一阶段：制作各个模块

**目标**：每个检测类型有一个能独立跑通的工具页。

**做什么**：
- 里氏硬度工具页（已完成）
- 锚杆抗拔工具页（已完成）
- 曲线绘制工具页（已完成）
- PDF 工具页（已完成）
- Word 转 PDF 工具页（已完成）
- 钻芯法工具页（待做）
- 回弹法工具页（待做）
- 内置模版制作器（待做）

**为什么是这个顺序**：先把能独立产生价值的功能做出来。一个内业员即便没有完整流程，也能用单个工具页完成某一项工作（比如只画曲线、只转 PDF）。

**用户怎么用**：打开软件 → 左侧文件树选数据 → 点对应工具页 → 右侧填参数 → 点运行 → 出结果。一个工具页一个工具页地用。

---

## 五、第二阶段：人工按流程出一份完整报告

**目标**：把第一阶段做好的模块串成一条线，用户手动走完一个完整流程。

**流程**：
```
1. 创建项目工作区
2. 上传原始数据 Excel
3. 选检测类型 → 跑计算 → 得到判定结果
4. 选模版 → 跑曲线 → 曲线图 + 数据填入报告 docx
5. 预览报告 → 确认签字栏 → 导出 PDF
```

**每步都是人点的**。程序不自动跳下一步——因为每步跑完人要看结果对不对。

**第一阶段和第二阶段的关系**：第一阶段造零件，第二阶段把零件拼成流水线。工具页仍然独立可用，但多了一个「项目模式」的入口。

**用户怎么用**：打开软件 → 新建项目 → 按流程一号一二三四走下去 → 出完整报告 → 归档。

---

## 六、第三阶段：引入 AI Agent

**目标**：人变成决策者，AI Agent 替代手工操作。

**核心变化**：

```
第二阶段：人点按钮 → 程序执行 → 人看结果 → 人点下一个按钮

第三阶段：人说一句话 → AI Agent 拆成步骤 → 调程序执行 →
         中间结果推给人 → 人做判定 → AI 继续 → 最终出报告
```

**用户怎么用**：
1. 打开软件，新建项目
2. 说：「这批数据是锚杆抗拔，用 GB 50086，模版用上次那个」
3. Agent 自动：读 Excel → 跑计算 → 生成曲线 → 填报告
4. Agent 推送中间判定结果：「3 号锚杆弹性位移量超限，不合格。其他 47 根合格。继续？」
5. 人说「继续」→ Agent 出完整报告并转 PDF
6. 人只做了两次决策：接受判定、确认继续

**为什么这是第三阶段而不是第二阶段就做**：Agent 需要每条工具线都稳定可靠，不能边修 bug 边让 Agent 调。第二阶段把人工流程跑通了，才敢把方向盘交给 Agent。

**Agent 能做什么、不能做什么**：

| Agent 能做 | Agent 不能做（必须人来） |
|-----------|----------------------|
| 读 Excel 选 sheet 选列 | 判断原始数据本身有没有问题 |
| 按规范跑计算 | 决定用哪个规范（选错规范 = 工程事故） |
| 套模版填报告 | 审核最终报告内容 |
| 转 PDF 归档 | 签字（物理签名或电子签章） |
| 报错时自动重试 | 碰到没见过的异常——必须停下等人 |

---

## 七、第四阶段：打包发布 + 持续填充

**目标**：稳定版本打包成安装包，持续加新检测类型。

**打包**：一个 exe 安装包，双击装好就能用。不依赖 Python 环境、不依赖 .NET SDK。

**持续填充**：
- 更多检测类型（钻芯、回弹、超声、静载……）
- 更多规范（JGJ 106、CECS 03……）
- 更多甲方模板适配
- 规范更新（GB 标准修订后更新数据库）

**这个阶段没有终点。** 只要还有新的检测类型，就可以一直加。

---

## 八、完整产品形态（四阶段合起来）

```
开软件 → 新建项目 → 拖入数据
    ↓
[阶段1] 选工具页 → 单独跑某个功能（画图/转PDF/算锚杆）
    ↓
[阶段2] 走完整流程 → 人工点七步出完整报告
    ↓
[阶段3] 说一句话 → AI Agent 替人操作 → 人只管决策
    ↓
[阶段4] 安装包双击装好 → 新项目类型不断加入
```

**最终用户体验**：装机 5 分钟，新手 10 分钟出第一份报告，老手一句话出报告。
