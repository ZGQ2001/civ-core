# Word 模板制作 SOP

civ-core 用 OpenXML SDK 处理用户的 Word 模板。**模板视觉零变化**——用户只需在原模板里加占位符。

> **重要（机制已变）**：旧的 `[[每根锚杆]]` / `[[批次]]` marker 克隆机制**已删除**，不再有「锚杆专用模板」。
> 现在锚杆和防火涂层**同一套机制**：数据表由程序按规范现建，模板里只放一个 `{{表格:xxx}}` 占位符。

## 第一步：让用户激活「模板助手」

civ-core 内置 `template_helper` 工具（ActivityBar 里）。**优先指引用户去那里**，不要在聊天里手把手猜：

```
1. 让用户在 ActivityBar 选「模板助手」
2. 选他想用的 Word 模板（.docx）
3. 选 catalog（一般 = "锚杆检测"）
4. 点「验证」
```

验证后给出反馈：matched（命中字段）/ unrecognized（写了但 catalog 找不到）/ unused（catalog 有但没用）/ hints（层级警告）。
注：`{{表格:锚杆}}` / `{{表格:防火涂层}}` 不是 catalog 字段，会落在 unrecognized——这正常，它们是数据表占位符，不是错误。

## 模板语法（薄壳 + 占位符）

| 语法 | 干什么 | 示例 |
|------|-------|-----|
| `{{字段名}}` | 项目信息占位符替换 | `委托单位：{{委托单位}}` |
| `{{img:xxx}}` | 嵌入图片（程序建表内部用，模板一般不用手写） | `{{img:曲线图}}` |
| `{{表格:锚杆}}` | 程序在此建**逐根 表2.4**（含曲线图）插入 | 独占一段放表的位置 |
| `{{表格:防火涂层}}` | 程序在此建**防火涂层检测结果表**插入 | 多类型组装时加 |

**字段名识别**：catalog key（`client_name`）/ 中文名（`委托单位`）/ 别名都识别。

## 核心心智模型：薄壳 + 数据表占位符

模板 =「薄壳」（封面 / 项目信息主表 / 结论 / 页眉页脚，全是 `{{}}` 项目字段）+ 一两个 `{{表格:xxx}}` 占位符。

- **数据表内部格式固定在代码里**（判定数据必须规范统一，判错=事故）——用户**不用、也不该**在模板里手画那张逐根表，或往里塞 `{{锚杆编号}}`/`{{0.1Nt位移}}`/`{{判定结果}}` 等行字段。那些由程序填。
- 用户只负责：薄壳里的**项目级（report 级）**字段 + 放 `{{表格:xxx}}` 的位置。
- 各批**灌浆日期**会自动出现在该批锚杆表的「灌浆日期」格（程序按批填值，无需 marker）。

## 模板结构示意

```
项目封面页：{{委托单位}} {{工程名称}} {{报告编号}}
═══════════════════════════════════
（章节文本，项目级一次）：受 {{委托单位}} 委托，本次对…

{{表格:锚杆}}          ← 独占一段。程序建逐根 表2.4（表2.4-1…N，曲线图嵌好）
{{表格:防火涂层}}      ← 可选：多类型组装时加这一段

═══════════════════════════════════
检测结论：{{检测结论}}    检测人员：{{检测人员}}
```

## 引擎行为关键点（C# DocxReportAssembler / AnchorWordTable / CoatingWordTable）

1. **Body 顶层段落**：`{{表格:xxx}}` 必须独占顶层段落，**不能在表格单元格内**——引擎在该段后插表、再删占位符段。
2. **没勾的清掉**：模板写了 `{{表格:防火涂层}}` 但本次没提供防火涂层数据 → 该占位符段自动删除，不留痕。
3. **缺占位符报错**：提供了某类型数据但模板缺对应 `{{表格:xxx}}` → 生成报清晰错误（提示去补占位符）。
4. **表标题编号**：单根锚杆→「表2.4」，多根→「表2.4-1 / -2 …」；节号在「报告填充」里可改（缺省 2.4）。
5. **图片**：逐根表内部的 `{{img:曲线图}}` 由程序按 anchor_id 在 `curveImageDir` 下找 svg/png/jpg 自动嵌入；没配目录 → 列入 `missingImages` 警告（不阻断）。

## 多检测类型一键报告

一份模板同时含 `{{表格:锚杆}}` + `{{表格:防火涂层}}` → 在「报告填充」勾「同时出防火涂层表」→ 选锚杆**结果 Excel** + 防火涂层**测点 Excel** → 一键出一份 Word（走 `report.assemble`）。

## 常见错误

| 现象 | 原因 |
|------|-----|
| 生成报「缺占位符 {{表格:锚杆}}」 | 模板里没写 `{{表格:锚杆}}`，或写进了表格单元格内。要独占顶层段落 |
| `{{某字段}}` 留原文 | catalog 里没这字段名/别名。用模板助手查"未识别"清单（`{{表格:xxx}}` 除外） |
| 数据表没出现 / 出现两次 | `{{表格:xxx}}` 不在 Body 顶层独占段，或写了多个同名占位符 |
| 图片没嵌入 | `curveImageDir` 没传 / 文件名跟 anchor_id 对不上。看 `missingImages` 警告 |
| 想手画逐根表 | 不需要——逐根表程序按规范建。模板只放 `{{表格:锚杆}}` 一段 |

## 验证流程（交付前必跑）

1. 模板助手 → 验证 → 项目字段 0 error-level hints（`{{表格:xxx}}` 落在 unrecognized 属正常）
2. report_generator 工具 → 选模板 + 数据 → 跑一次小样本
3. 用 Word 打开输出 docx，肉眼检查：项目字段全替换（无 `{{` 残留）、逐根表数量/编号对得上、曲线图嵌入到位、没勾的类型占位符已清
