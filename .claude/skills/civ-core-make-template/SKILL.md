# Word 模板制作 SOP

civ-core 用 OpenXML SDK 处理用户的 Word 模板。**模板视觉零变化**——用户只需在原模板里加占位符 + 成对 marker 段。

## 第一步：让用户激活「模板助手」

civ-core 内置了 `template_helper` 工具（ActivityBar 里）。**优先指引用户去那里**，不要在聊天里手把手猜：

```
1. 让用户在 ActivityBar 选「模板助手」
2. 选他想用的 Word 模板（.docx）
3. 选 catalog（一般 = "锚杆检测"）
4. 点「验证」
```

验证后给出 4 类反馈：
- ✅ matched：占位符 → 字段（能命中的）
- ❌ unrecognized：写了但 catalog 找不到
- ⚠️ unused：catalog 有但模板没用
- 🔔 hints：**level mismatch 警告**（字段层级错了）

## 模板语法（4 种 marker）

| 语法 | 干什么 | 示例 |
|------|-------|-----|
| `{{字段名}}` | 占位符替换 | `委托单位：{{委托单位}}` |
| `{{img:xxx}}` | 嵌入图片（按 anchor_id 自动拼路径） | `曲线图：{{img:曲线图}}` |
| `[[每根锚杆]]...[[/每根锚杆]]` | 段内元素克隆 N 次 | 包住表格 + 标题段 |
| `[[批次]]...[[/批次]]` | 外层批次克隆 | 包住整个批次段（含内层 `[[每根锚杆]]`） |

**字段名识别**：catalog key（`grouting_date`）/ 中文名（`灌浆日期`）/ 别名（`注浆日期`）都识别。

## 字段层级 vs marker 位置（**最容易踩**）

每个字段在 catalog 里有 `level`，4 种：

| Level | 例 | 该放哪 |
|-------|-----|-------|
| `report` 报告级 | 甲方 / 工程名 / 监理 / 报告编号 | **任何位置**（外层最好） |
| `detection_item` 检测项目级 | 检测项目 / 检测依据 | 检测项目段（暂未启用） |
| `batch` 检测批级 | **灌浆日期** / 工程参数 | **必须**在 `[[批次]]...[[/批次]]` 内 |
| `component` 构件级 | 锚杆编号 / 位移 / 判定结果 | **必须**在 `[[每根锚杆]]...[[/每根锚杆]]` 内 |

**level mismatch 后果**：
- 把 `component` 字段放到外面 → 引擎找不到值 → 该位置渲染失败 / 留原文
- 把 `report` 字段放到 `[[每根锚杆]]` 内 → 每根锚杆都重复一遍同样的值（浪费 + 视觉吓人）

**模板助手会自动检测这种错误并给出修复建议。**

## 嵌套结构（多批次报告）

```
项目封面页：{{委托单位}} {{工程名称}}
═══════════════════════════════════

[[批次]]
  本批批次号：{{batch_id}}    灌浆日期：{{灌浆日期}}
  
  [[每根锚杆]]
    表 2-{{锚杆序号}}：锚杆 {{锚杆编号}} 检测数据
    （插入数据表，含 {{0.1Nt位移}} {{弹性位移量}} {{判定结果}} 等）
    曲线图：{{img:曲线图}}
  [[/每根锚杆]]
[[/批次]]

═══════════════════════════════════
检测结论：{{检测结论}}      检测人员：{{检测人员}}
```

## 引擎行为关键点

1. **Body 顶层段落**：marker 必须放顶层独占段落，**不能在表格单元格内**
2. **成对必配**：`[[xxx]]` 必须配 `[[/xxx]]`，错配会抛清晰错误
3. **嵌套顺序**：`[[批次]]` 必须包 `[[每根锚杆]]`，反过来不行
4. **自动 dispatch**：模板有 `[[批次]]` → 走 `GenerateMultiBatch`；没有 → 走单批 `Generate` + grouting_date fallback（取第一批的值）
5. **图片占位符**：`{{img:曲线图}}` 自动按 anchor_id 拼 `<curveImageDir>/<anchor_id>.png`；找不到 → 列入 `missingImages` 警告

## 常见错误

| 现象 | 原因 |
|------|-----|
| `{{灌浆日期}}` 没替换 | 模板没用 `[[批次]]...[[/批次]]` 包住 → 走了 fallback 路径取第一批的值。要么加 marker，要么接受 fallback |
| `{{某字段}}` 留原文 | catalog 里没这字段名/别名。用模板助手查"未识别"清单 |
| 图片没嵌入 | `curveImageDir` 没传 / 文件名跟 anchor_id 对不上。看 `missingImages` 警告 |
| 每根锚杆出现重复的甲方名 | 把 report-level 字段放进了 `[[每根锚杆]]` 内。挪到外面 |
| "找不到行重复起始锚点" 报错 | `[[每根锚杆]]` 写在表格单元格里了。必须独占顶层段落 |

## 验证流程（交付前必跑）

1. 模板助手 → 验证 → 0 unrecognized + 0 error-level hints
2. report_generator 工具 → 选模板 → 跑一次小样本（1-2 个批次）
3. 用 Word 打开输出 docx，肉眼检查：
   - 占位符全部替换（无 `{{` 残留）
   - 批次/锚杆克隆数量对得上
   - 图片嵌入到位
