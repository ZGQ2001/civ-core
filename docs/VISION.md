# civ-core（筑核）产品愿景与实现路线

> 写给所有人——用户、开发者、AI——看的统一蓝图。说清楚这个软件要做什么、做到什么程度、怎么一步步到那。

---

## 一、产品定位

**一句话：把土木检测内业从 Excel 手工活变成「丢数据进去、报告出来」的一键流程。**

当前土木检测公司的内业工作流：

```
甲方送样 → 试验员做试验 → 记录原始数据（纸质/Excel）
    ↓
内业员手工：
  ① 把原始数据整理成标准表格
  ② 查规范条文判断合不合格
  ③ 画荷载-位移曲线
  ④ 填 Word 报告模板
  ⑤ 转 PDF 归档
    ↓
报告交给甲方
```

每一步都是人手工做。一个项目几十根锚杆，重复劳动量巨大。civ-core 把步骤 ②③④⑤ 自动化。

---

## 二、目标用户

| 用户 | 场景 | 诉求 |
|------|------|------|
| 检测公司内业员 | 每天处理多个项目报告 | 减少重复操作，降低出错率 |
| 项目负责人 | 审核报告格式和判定结果 | 格式规范统一，判定可追溯 |
| 试验员 | 记录原始数据 | Excel 模板方便填写 |

**非目标用户**：不做外业数据采集（保留给试验员的手持设备），不做项目管理（保留给公司 ERP）。

---

## 三、产品形态

- **桌面应用**（非 Web，非 SaaS）。Windows 单机运行。
- **离线可用**。不依赖网络，规范数据库本地存储。
- **一次配置，重复使用**。模版做好后，换项目只换数据文件。

---

## 四、核心工作流

```
┌──────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────┐
│ 输入数据  │ →  │  计算与判定   │ →  │  图表与报告   │ →  │  输出交付  │
└──────────┘    └──────────────┘    └──────────────┘    └──────────┘

输入数据：
  用户下载空白 Excel 模板 → 填入试验数据 → 上传

计算与判定：
  程序读取 Excel → 按规范（GB 50086 / JGJ 106 等）计算
  → 自动判定合格/不合格 → 输出结构化结果

图表与报告：
  位移数据 → matplotlib 曲线图（PNG）
  计算结果 → 填入报告模板（docx）
  曲线图 → 嵌入报告

输出交付：
  完整 Word 报告 + PDF 归档
```

## 六、技术决策记录

| 决策 | 原因 | 日期 |
|------|------|------|
| 前端 Vite + React + TS + Tailwind v4 | VSCode 风工业冷峻 UI，Qt 视觉天花板 | 2026-05-20 |
| 主进程 Tauri 2 | 比 Electron 轻量，Rust 性能 | 2026-05-20 |
| 双 Sidecar（Python + C#） | Python 有 matplotlib 无可替代；C# OpenXML/ClosedXML 原生强 | 2026-05-20 |
| 协议 JSON-RPC 2.0 over stdin/stdout | 极简，绕开 HTTP/WS IPC 复杂度 | 2026-05-20 |
| Python 仅保留 plot_curves | matplotlib 是 Python 生态的最后壁垒；其他全迁 C# 简化部署 | 2026-05-22 |
| 路由默认 C#，白名单 Python | 用户方向「以后代码都用 C#」 | 2026-05-22 |
| 报告模版用 Word docx + 绑定配置 | 甲方给的格式就是 Word；用内部编辑器绑定字段，不改代码换模版 | 2026-05-29 |

## 七、不做的事

- **不做 SaaS / Web 版**。桌面端就够了，离线可用是刚需。
- **不做手机端**。内业工作一定是电脑上做的。
- **不做 BIM / 三维**。这是内业报告工具，不是建模软件。
- **不做项目管理 / CRM**。那是 ERP 的事。
- **不接仪器直读**。原始数据录入保留给试验员（Excel 模板就是接口）。
- **不做云端同步**。文件在本地，用户自己决定放哪。

---

## 八、成功标准

> 一个内业员拿到新项目数据后，从打开软件到导出完整报告，操作步骤不超过：

1. 打开软件 → 工作区选项目文件夹
2. 上传数据 Excel → 选检测类型 → 点运行
3. 点「生成报告」→ 选模版 → 输出 docx
4. 点「转 PDF」→ 归档

**目标：4 步出报告。**
