# 审查报告：RPC Handler 输入校验缺口

**日期**：2026-05-24  
**审查项**：C# RPC handler 层的输入校验——文件不存在、数值越界、路径注入、空数据  
**审查方法**：逐 handler 审查所有入参 → 逐 calc 函数审查边界值处理

---

## 发现

### 🔴 锚杆物理参数无正值校验（AnchorHandlers.cs:153-171）

`ParseParamsByBatch` 解析 `P`（拉力）、`Lf`（自由段长度）、`La`（锚固段长度）、`A`（截面积）、`E`（弹模）：

```csharp
double p = ap.GetProperty("P").GetDouble();   // 可以是负数
double lf = ap.GetProperty("Lf").GetDouble(); // 可以是 0
// ... 全部无范围校验
```

如果前端传错或用户手填 Excel 填反了，传 `P=-200` 或 `A=0`，锚杆承载力计算会出负的弹性伸长量——物理上无意义的结果会直接写进报告 xlsx，且没有任何报错提示。

**风险**：出错的工程报告可能被提交到甲方。

### 🔴 PreviewExcel 无文件存在检查（LeebHandlers.cs:110）

```csharp
public static object PreviewExcel(JsonElement? @params) {
    var path = p.GetProperty("path").GetString() ...;  // 只取字符串
    using var wb = new XLWorkbook(path);  // 文件不存在→ClosedXML 抛异常
}
```

`LeebHandlers.Run` 的 `ReadWorkbook` 内部有 `File.Exists` 检查，但 `PreviewExcel` 没有。文件不存在时 ClosedXML 抛出的原始异常信息会直接暴露给用户。

### 🟠 路径遍历无防护（全局）

所有 handler 接受文件路径参数但都不校验路径是否包含 `../`、是否在允许范围内。当前前端限制了文件选择器只能选文件，但如果有人通过 RPC 直接调 `leeb.run` 传 `../../etc/passwd`，C# 侧照单全收。

### 🟠 GenerateTemplate 无输出目录检查（AnchorHandlers.cs:47）

```csharp
AnchorTemplateWriter.Write(outputXlsx, standard);  // outputXlsx 目录不存在→写失败
```

`AnchorTemplateWriter` 内部可能有检查，但 handler 层没做。

### 🟡 角度参数校验位置太深（LeebHandlers.cs:45 → LeebCalculator.cs:41）

`angleDegrees` 在 handler 层无校验，一直传到 `CalcSteel` 的第 41 行才检查 `{-90,-45,0,45,90}`。如果传 30°，用户看到的报错会包在 `-32603 Internal error` 里糊成一片，而不是清晰的"测量角度只支持 -90/-45/0/45/90 度"。

## 评级

| 问题 | 严重度 | 影响 |
|------|--------|------|
| 锚杆物理参数无正值校验 | 🔴 | 可能产出错误工程报告 |
| PreviewExcel 无文件检查 | 🔴 | 用户看到 ClosedXML 原始报错 |
| 路径遍历无防护 | 🟠 | 安全风险（需前端配合才触发） |
| GenerateTemplate 无目录检查 | 🟠 | 写文件可能静默失败 |
| 角度校验位置太深 | 🟡 | 报错信息被 Internal error 糊掉 |

---

## 建议修复

1. **AnchorParams.Create** 加 `> 0` 校验，P/A/E 必须正数
2. **PreviewExcel + GenerateTemplate** 调用前加 `File.Exists` / `Directory.Exists` 守卫
3. **所有文件路径参数** 加 `Path.GetFullPath` + 限制在工作区范围内
4. **angleDegrees** 在 handler 入口处就校验，给用户清晰的错误提示
