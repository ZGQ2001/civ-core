# 审查报告：前端按钮防重复点击 & Loading 状态

**日期**：2026-05-24  
**审查项**：用户双击按钮是否会触发并发请求——所有工具页 Run 按钮 + 文件操作按钮 + 预设管理按钮  
**审查方法**：逐页检查 disabled/running 守卫

---

## 发现

### 🟢 Run 按钮全部有守卫（4/4 工具页）

| 工具 | 守卫 | 方式 |
|------|------|------|
| plot_curves | `canRun = ... && !c.running` | `disabled={!canRun}` ✅ |
| data_processing | `canRun = ... && !c.running` | `disabled={!canRun}` ✅ |
| pdf_tools | `canRun = ... && !c.running` | `disabled={!canRun}` ✅ |
| word2pdf | `canRun = ... && !c.running` | `disabled={!canRun}` ✅ |

加上 data_processing「生成模板」按钮也有 `disabled={...kind === 'running'}` 守卫。**核心 Run 路径全部保护到位。**

### 🟠 plot_curves 预设操作按钮无 running 守卫（6 个）

`plot_curves/Page.tsx` 的预设管理按钮：

| 按钮 | 行号 | 守卫状态 |
|------|------|----------|
| 保存 | 598 | ❌ 无守卫 — 狂点可能多次 savePreset RPC |
| 新建 | 615 | ❌ 无守卫 — 创建多条同名空白曲线 |
| 复制 | 617 | ❌ 无守卫 |
| 重命名 | 621 | 仅 `disabled={!isUser}` — 不防双击 |
| 删除 | 627 | 仅 `disabled={!isUser}` — 不防双击 |

`handleSave`、`handleCopy`、`handleDelete` 等触发异步 RPC 操作（`savePreset`、`copyPreset` 等），但没有 loading 状态阻止重复提交。用户连续点保存 → 多份相同的预设副本。

### 🟠 FileTree 操作无异步守卫（3 个）

| 操作 | 行号 | 问题 |
|------|------|------|
| 删除 (`confirmDelete`) | 538 | `setDeleteTarget(null)` 在前 → 第二次点击被挡，但仅在第一次 fn 入口后 |
| 粘贴 (`doPaste`) | 576 | ❌ 无守卫，连点多次粘贴 → 多个 `files.copy` RPC |
| 撤销删除 (`doUndoDelete`) | 554 | ❌ 无守卫，连点多次 |

`confirmDelete` 有部分保护（函数入口设 null），但 `doPaste` 和 `doUndoDelete` 完全没有——依赖的是 React 的合成事件不能在同一事件循环中触发两次点击，但物理双击（两次独立 click）可以绕过。

### 🟢 App 级操作有守卫

`App.tsx` 的新建工作区和打开文件夹操作虽无 running 状态，但使用原生 `openDialog`（模态对话框）天然阻止重复操作。

---

## 评级

| 问题 | 严重度 | 触发难度 |
|------|--------|----------|
| Run 按钮 | 🟢 | — 全部有守卫 |
| 预设保存/复制/删除 | 🟠 | 中等 — 快速双击触发 |
| FileTree 粘贴/撤销 | 🟠 | 中等 — 快速双击触发 |
| 生成模板按钮 | 🟢 | — 有守卫 |

---

## 建议修复

1. **plot_curves 预设操作**：加 `saving`/`deleting` 状态，保存中禁用所有预设按钮
2. **FileTree 操作**：加 `operationInProgress` 状态，异步操作期间禁用删除/粘贴/撤销按钮
3. **统一模式**：提取 `useAsyncGuard` hook（`{ running, run: () => Promise }`），替代散落的 `try/catch`
