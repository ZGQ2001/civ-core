# 审查报告：点击顺序错误 & 操作流程守卫

**日期**：2026-05-24  
**审查项**：没选文件点计算、计算中切换参数、算完换工具——每一步的错误操作是否有拦截  
**审查方法**：逐 controller 审查前置条件校验 + 运行中状态锁 + stale request 处理

---

## 发现

### 🟢 前置条件校验全部到位（4/4 工具）

所有工具页的 Run 按钮在缺少必要前提时都被 `canRun` 禁用：

| 工具 | 前置条件 | 方式 |
|------|----------|------|
| plot_curves | 必须选 Excel + 选曲线 | `canRun = ...excelPath && ...preset && !running` |
| data_processing | 必须选 Excel | `canRun = ...excelPath && !running` |
| pdf_tools | merge 需 ≥1 文件 + 输出路径 / split 需文件 + 输出目录 + 表达式 | `canRun` 按 mode 区分 |
| word2pdf | 需 ≥1 文件 + 输出目录 | `canRun = !running && inputs.length >= 1 && ...outDir` |

此外 `data_processing.run()` 内部还有 `if (!excelPath || running) return null` 二次兜底。

### 🟢 stale request 有竞态处理

`data_processing` 的 Excel 预览使用 `reqIdRef` 递增 + `.then` 内 `if (myId !== reqIdRef.current) return` 模式——用户快速切换 sheet 时不会出现旧请求覆盖新请求的数据混乱。✅

### 🟢 参数修改有警告提示

plot_curves 修改曲线参数后显示黄色警告「曲线已被调参（运行 / 预览均用编辑版）」+「还原」按钮。防止用户改了参数却忘了还原就运行。✅

### 🟠 计算中切换模式/类型——无中断机制（4 个工具）

| 场景 | 行为 | 风险 |
|------|------|------|
| data_processing: leeb 计算中切换到 anchor | 旧计算继续跑，结果写旧的 state | 界面显示「计算中…」但用户已经在锚杆设置页——等旧结果回来时弹到错误位置 |
| pdf_tools: merge 计算中切换到 split | 同上 | merge 结果回来但 UI 已在 split 模式——结果不会显示在正确位置 |
| plot_curves: 出图中切换曲线 | 旧曲线继续出，`c.running` 挡住新运行 | 无法中止正在进行的批量出图 |
| word2pdf: 转换中添加/删除文件 | running 期间 `canRun` 挡住重跑，但文件列表可以改 | 用户可能以为改了列表会生效 |

**根因**：4 个 controller 的 `run()` 没有 abort 能力。`setRunning(true)` 是单向的——只能等 finally 里 `setRunning(false)`。没有 `cancel()` 方法，没有 AbortController。

### 🟡 运行中允许修改参数——可能产生困惑

`running` 期间用户可以：
- 切换 sheet / 改 headerRow / 改 outputPath（data_processing）
- 切换预设 / 改 DPI / 改图例位置（plot_curves）
- 添加/删除文件列表（pdf_tools / word2pdf）

虽然 `running=true` 阻止了再次点击 Run，但修改的参数不会影响当前运行（因为参数已传给 RPC），用户可能以为改了参数就能影响正在进行的计算。

### 🟢 空数据处理 OK

`LeebExcelReader.ReadComponents` 跳过完全空行（A 列和 C 列都空→continue），返回 `components.Count == 0` 时上层丢弃该 batch。不会因空数据崩溃。

## 评级

| 问题 | 严重度 | 说明 |
|------|--------|------|
| 前置条件校验 | 🟢 | 全部到位 |
| stale request 处理 | 🟢 | reqIdRef 模式成熟 |
| 计算中切换无中断 | 🟠 | 无法中止慢计算 |
| 运行中允许改参 | 🟡 | 可能让用户困惑 |
| 空数据处理 | 🟢 | Excel reader 层面已处理 |

---

## 建议

1. **加 abort 能力**：controller 加 `cancel()` 方法 + AbortController，切换模式/类型时自动 cancel 当前运行
2. **运行中锁定参数**：`running=true` 时禁用 Excel/sheet/预设切换控件（至少加灰色 disabled 样式）
