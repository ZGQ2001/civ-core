## 1. 背景与目标

**现状**：防火涂层装配线 L1–L4 已落（算 + 出「数据分析」宽表 xlsx，见 `CoatingAnalysisSheet`），但**止步于结果 Excel，不能出交付用的报告表**。我原本推荐补 Layer 5 走 Word 模板路线（catalog + RowResolver + 扩 marker 引擎）。

**用户纠偏**：检测内业的真实交付流程是 **数据 → xlsx 报告表（固定格式）→ 复制进 docx 报告**。所以：
- 程序该产出的是**那张固定格式的报告表 xlsx**，不是直接拼 Word。
- 报告表格式是固定的，照 `报告表格.xlsx` 来。
- 用户同时意识到「**锚杆那条也走了弯路**」——锚杆现在走 Word `[[每根锚杆]]` marker 引擎，其实它的交付物也是一张固定 Excel 表（见 §6）。

**目标**：做一个「**报告表生成器**」，按检测类型 + 涂层类型，把计算结果产出成可直接贴进 docx 的固定格式报告表 xlsx。本轮覆盖**防火涂层**；架构预留通用化，锚杆**本轮不动**（见 §6）。

## 2. 关键发现：`报告表格.xlsx` 里的固定格式

逐格 dump 后，这个文件含 **5 张**报告表（= 检测内业实际交付的成形表）：

### ① 膨胀型（sheet「防火涂层国标薄型、超薄型、防腐涂层」）← 用户说要改的那张
```
序号 | 构件编号 | (1处)平均值(μm) | (2处)平均值 | (3处)平均值 | (4处)平均值 | (5处)平均值 | 平均值(μm) | 设计值(μm) | 检测结果
 1   | 地上一层A×1轴 |  232 | 230 | 230 | 231 | 230 | 231 | 220 | 合格
```
- **一构件一行**。列 = (1处)~(5处) 的**平均值** + 总平均 + 设计值 + 检测结果。
- 即：录入是 5 处 × 每处 3 测点 = 15 个原始值；报告表只放每处的均值（5 个）。

### ② 厚型·钢梁（sheet「防火涂层其他，钢梁」）
```
序号 | 构件位置 | 测点 | 实测涂层厚度(μm)[梁侧面/梁侧面/梁底面] | 平均值(μm)
 1   | …(27~28)轴钢梁 | 截面1 | 3825 | 3982 | 4605 | 3916.5(构件均值,合并)
                       | 截面2 | …(5 截面)
```
- 一构件跨 N 截面行，列 = 3 个面的实测值 + 构件平均（合并）。**无设计值列、无判定列。**

### ③ 厚型·钢柱（sheet「防火涂层其他，钢柱」）
- 同 ②，4 个面（东/西/南/北侧面），3 截面，单位 **mm**（梁是 μm）。同样无设计/判定列。

### ④ 锚杆（sheet「锚杆」）
- 一锚杆一张表，全是 `{{锚杆编号}}`…`{{弹性位移量 M}}`…`{{img:曲线图}}` 占位符。**正是用户说"走弯路"的那张**——它本质也是固定 Excel 表（详见 §6）。

### ⑤ 里氏硬度（sheet「里氏硬度」）
- 里氏硬度抗拉强度结果表，9 列硬度读数 + 抗拉特征值。本轮不做，登记备查。

## 3. 生成机制：已定 D（母版管样式 + 表头认列 + 代码建合并，无占位符）

用户连环追问后定的最终形态。核心是把「样式」和「数据映射」分开放，并且**不让用户维护任何占位符**：

| 关注点 | 谁负责 | 改了会怎样 |
|---|---|---|
| **外观**：字体宋体+Times、边框、列宽、表头文字、底色 | **母版 xlsx** | 在 Excel 里改母版，**零代码、零发版** |
| **取数→落格**：哪列填什么 | **代码按表头文字认列**（复用 `CoatingSheetUtil.ReadHeaderMap`/`RequireColumn`，读测点数据早就这么干）| 表头含关键词就自动认到；认不到留空，绝不错位 |
| **结构**：每构件几行、序号/位置/均值跨该构件所有行合并 | **代码**（稳定规律，写一次不动）| 不随外观变 |
| **行数**：截面数 | **数据**（⌈长度/间距⌉）| 自动 |

机制要点：
1. `File.Copy(母版 → 输出)`：输出即母版逐字节副本，**宋体+Times New Roman / 边框 / 合并 / 列宽零损耗**。
2. 母版那张 sheet 只留**表头行 + 一行空白「样式参考行」**（不预设数据行的合并，免得运行时拆来拆去）。
3. 代码读表头建「表头文字→列」映射；每构件按截面数克隆样式行、按表头填值；厚型补构件级合并（序号/位置/均值跨该构件所有行）。
4. **选哪张 sheet 不写死**：母版加一张「格式映射」配置 sheet（`标准, 涂层类型, 构件类型 → sheet 名`），代码读它选表。新增构件类型/甲方版式 = 加母版 sheet + 加一行映射，纯配置。
5. 加 Verify snapshot 把输出结构钉死，换版式时 diff 可见。

**为什么不是 B（代码画格子）**：B 把样式焊在代码里，连改个字体/边框都要改代码 + 重打包发版（二进制用户还得等更新）。D 把外观挪进母版，外观改动零代码。
**为什么不是「占位符母版」**：那要用户维护 `{{}}` 占位符——是额外负担，用户明确否了。D 的母版就是张「数据行留空的报告表」，没有占位符。

> 唯一前置工作：把 `报告表格.xlsx` 改造成母版——每格式留表头 + 一行样式参考行 + 一张映射 sheet（无占位符，就是张空白报告表）。一次性、Excel 里就能做（可由我产出一版供你核对，见 Q8）。

## 4. 报告表 ↔ 数据映射

### 4.0 选表 + 单位规则（已定）

| 标准 | 涂层类型 | 格式/布点 | 单位（仪器）| 母版 sheet |
|---|---|---|---|---|
| 国标 | 薄/超薄（膨胀型）| 格式① **5处×3点** | μm（涂层测厚仪）| 防火涂层国标薄型、超薄型、防腐涂层 |
| 国标/地标 | 厚型 | 格式②③ 截面×面 | mm（游标卡尺）| 防火涂层其他·钢梁/钢柱 |
| 地标 | 薄/超薄（膨胀型）| 截面×面（沿用现状）| μm（涂层测厚仪）| 防火涂层其他·钢梁/钢柱 |

- **单位由涂层类型决定**：厚型 mm、薄/超薄 μm。内部统一 mm，写表时按目标格式换算（μm = mm×1000）。
  与母版自洽：钢梁样本 ~3.8mm = 薄型→μm，钢柱 ~26mm = 厚型→mm。
- 格式②③ 按**构件类型**分（梁 3 面 / 柱 4 面）；同布局下单位随涂层类型（厚 mm / 薄 μm）。
  → 母版按「格式×单位×构件类型」需要几张配几张，由「格式映射」sheet 索引；缺的组合用户加一张母版 sheet 即可（零代码）。

数据全部可从现有 `CoatingMemberInput` + `CoatingMemberResult` + `Points`（按 `SectionNo`/`Position` 分组）推导，**无需改判定核心**。

### 膨胀型（格式①）
| 报告表列 | 数据来源 |
|---|---|
| 序号 | 构件序号 |
| 构件编号 | `CoatingMemberInput.Location` |
| (N处)平均值 | `Points` 按 `SectionNo==N` 分组取均值（每处 3 测点）|
| 平均值 | `CoatingMemberResult.MeanThickness` |
| 设计值 | `CoatingMemberInput.DesignThickness` |
| 检测结果 | `CoatingMemberResult.Verdict`（合格/不合格）|

### 厚型（格式②③）
| 报告表列 | 数据来源 |
|---|---|
| 序号 / 构件位置 | 构件序号 / `Location`（合并）|
| 测点（截面 N）| `SectionNo` |
| 实测厚度[面名] | `Points` 按 `(SectionNo, Position)` |
| 平均值 | `CoatingMemberResult.MeanThickness`（构件级合并）|

> 厚型母版②③ 无设计值/判定列 → **照母版，不加**（用户「都按母版来」）。判定结论若需要，走报告正文/另表，不进这张数据报告表。

## 5. 膨胀型录入网格修正（用户明确要改）

> 这是 `coating.expand_template` 生成的**录入网格**（用户填数字的「测点数据」sheet），与 §4 的报告表是两回事；不论报告表走哪种机制，这个网格都要先改对。

**口径（用户拍板）**：膨胀型分国标/地标。**国标膨胀型按「防腐涂层」口径 = 5 处，每处 3 点**（参照 GB/T 50621 §12 防腐）；地标膨胀型仍按截面布点（沿用现状）。之前被错改成"5 测点×重复读数"。

**现状（错）**：`CoatingTemplateExpander` 国标膨胀型分支生成「索引列=测点号 1~5，数据列=第一次/第二次/第三次」（误解成"5 测点各测 3 次重复读数"）。

**应为**：「索引列=**处号（第一处~第五处）**，数据列=**测点1/测点2/测点3**」（5 处空间布点，每处 3 个测点）。与报告表格式①的 (1处)~(5处) 对齐。

涉及改动：
- `CoatingTemplateExpander.FivePointHeaders`：`{第一次,第二次,第三次}` → `{测点1,测点2,测点3}`。
- 索引列名 `CoatingColumns.PointNo`「测点号」→「处号」（或值用"第一处…"，需 Q3 定）。
- `CoatingAnalysisSheet` 膨胀型分支的表头（`测点号`/`测点均值`/复用 `FivePointHeaders`）同步。
- `CoatingExcelReader`：现读法是"索引列 + 右侧非空列"通用读，**改列头/索引名不破坏读取**（已认 `截面号`/`测点号` 两者；若索引名改"处号"要加进 `RequireAnyColumn` 白名单）。
- 判定核心 `CoatingMath` 不变（均值 = 15 点均值，与分组无关）。
- vault `涂层厚度检测.md` 膨胀型伪代码同步；核对 GB/T 50621 §12 + 用户口径（用户为领域权威，已明确"第一处~第五处 × 测点1/2/3"）。
