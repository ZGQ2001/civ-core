# 占位符模板速成

> 给报告模板填字段最简单的方式：在 Word 里写 `{{字段名}}`，程序自动替换成实际值。
> 直接编辑 .docx，不用切到任何配置工具。

---

## 1. 怎么用

**步骤 1**：拿一份甲方给的 Word 报告模板（`.docx`）

**步骤 2**：在需要填值的位置打 `{{中文字段名}}` 或 `{{snake_case_key}}`，比如：

```
锚杆编号：{{anchor_id}}        ← 用 Key
弹性位移：{{弹性位移量}}        ← 用中文名（推荐 — 跟规范用词一致）
最大试验荷载下弹性位移量：{{最大试验荷载下弹性位移量}}    ← 别名（catalog 里映射到 elastic_displacement）
判定结果：{{判定结果}}
```

**步骤 3**：保存模板，打开「报告填充」工具页（左侧 Activity Bar 第 3 个图标）→ 选好输入 Excel + 模板 + 填项目信息 → 点「生成 Word 报告」

引擎会扫整个 Word：每一段、每一个表格里的每一个段落，把 `{{xxx}}` 替换成对应值。

---

## 2. 语法约定

### 文本占位符

`{{key}}` 或 `{{中文名}}` 或 `{{别名}}` —— 都识别。优先级：ASCII key 直查 > 中文名/别名反查。

数值字段按 catalog 的 `DefaultFormat` 格式化（如弹性位移量是 "0.00" → 2 位小数），不会输出 `1.23456789` 这种裸 double。

### 图片占位符

`{{img:xxx}}` —— 加 `img:` 前缀。resolver 返回的值当文件路径，引擎自动嵌入 OpenXML Drawing。

锚杆模板里的 `{{img:曲线图}}` 配合「报告填充」工具的「曲线图目录」picker，引擎按 anchor_id 拼 `{目录}/{锚杆编号}.png` 嵌入。

文件不存在 / 路径解析失败 → 留原文 + 生成结果里 `missingImages` 列出来，前端会显示警告。

### 数据表占位符（程序按规范建表）

数据表**不用自己在 Word 里画**——在要放表的位置写一段独占的占位符，程序按规范格式现建：

```
{{表格:锚杆}}        ← 锚杆：逐根「表2.4 单根结果表」（含曲线图），程序建好插这里
{{表格:防火涂层}}    ← 多类型组装时再加：防火涂层检测结果表
```

引擎规则：
- **占位符段必须独占一段**、在正文顶层（不能塞在表格单元格内）
- 程序在该处插入按规范建好的表（锚杆逐根一张 表2.4、含 `{{img:曲线图}}` 自动嵌图）
- 表标题编号按公司标准：单根→「表2.4」，多根→「表2.4-1 / 2.4-2 …」（节号在「报告填充」里可改，缺省 2.4）
- 模板里写了 `{{表格:防火涂层}}` 但本次没勾防火涂层 → 该占位符自动清掉，不留痕
- 提供了某类型数据但模板缺对应占位符 → 生成报清晰错误（提示去补占位符）

> 注：逐根表的**内部格式固定在代码里**（判定数据必须规范统一）——你不用、也不该在模板里手画那张表或往里塞 `{{锚杆编号}}` 等行字段；那些由程序填。模板里只需放 `{{表格:锚杆}}` 一段 + 封面/项目信息/结论那层的 `{{}}` 占位符。

## 3. 锚杆抗拔（anchor）可用字段

> 中文名 / 别名都能直接当占位符用，引擎按 catalog 反查 Key。

完整字段在「报告填充」工具页底部的「模板可用占位符清单」按钮（49 个），按 4 组展示：工程参数 / 原始数据（每根锚杆）/ 计算结果（每根锚杆）/ 用户输入（项目级）。

> 其中「工程参数 / 原始数据 / 计算结果」这些**每根锚杆级**字段已由 `{{表格:锚杆}}` 自动建表填入——模板里**一般只手写「用户输入（项目级）」那组**到封面/总览/结论。下表的字段映射主要用于理解程序建表时的取值口径。

几个关键约定：

| 模板写 | 实际命中 | 单位换算 |
|--------|---------|---------|
| `{{轴向拉力设计值}}` | `axial_design_load_kn`（派生）| 内部计算用 N，输出 kN（÷1000）|
| `{{0.1Nt位移}}` | `disp_01nt`（别名）| —— |
| `{{1.2Nt位移}}` | `disp_12nt_5min`（持荷 5min 取最稳定）| —— |
| `{{杆体弹模}}` | `elastic_modulus`（别名）| —— |
| `{{允许值上限}}` / `{{允许值下限}}` | `upper_limit` / `lower_limit` | —— |
| `{{最大试验荷载下弹性位移量}}` | `elastic_displacement`（别名）| —— |
| `{{试验人员}}` / `{{试验日期}}` | `inspection_engineer` / `inspection_time`（别名）| 已合并到检测人员/检测时间 |
| `{{锚杆序号}}` | `anchor_index`（引擎自动注入 1/2/3）| —— |

## 4. 常见坑

### 占位符跨 Run

Word 输入法常把 `{{anchor_id}}` 拆成多个 Run（"{{" / "anchor_id" / "}}"）—— 引擎在段落级先合并 Run 文本再替换，所以肉眼看到完整占位符就 OK。

代价：同段落内多种字体差异会丢，只保留首 Run 的字体（极少见，可接受）。

### 占位符之间不要跨段

`{{anchor_id}}` 不能跨段落（不能在 `{{` 后 `Enter` 换行）。`{{` 和 `}}` 必须在同一段落里。

### 表格占位符段不能在表格内

`{{表格:锚杆}}` / `{{表格:防火涂层}}` 必须独占段落且在 Body 顶层，不能塞在表格单元格里——引擎按段落顶层位置找，在该段后插入建好的表、再删掉占位符段。

### 未识别的占位符留原文

拼错的占位符会留原文 + 加到 `unknownKeys` 列表，前端能看到提示。这是设计——不阻断生成，让用户能见到没填上的占位符。

## 5. 完整模板结构示意（锚杆抗拔报告）

```
─────────────────────────────────────────────
封面表（项目级一次）：
  报告编号：{{报告编号}}
  委托单位：{{委托单位}}
  项目名称：{{项目名称}}
  检测类别：{{检测类别}}

总览表（项目级一次）：
  检测仪器：{{检测仪器1}} {{检测仪器2}}
  检测时间：{{检测时间}}
  检测人员：{{检测人员}}
  检测结论：{{检测结论}}
  ...

章节文本（项目级一次）：
  受 {{委托单位}} 委托，本次对该项目边坡的 N 根锚杆...

─────────────────────────────────────────────
{{表格:锚杆}}        ← 独占一段。程序在此建逐根 表2.4（含曲线图），表内字段全自动填
{{表格:防火涂层}}    ← 可选：多类型组装时加这一段
─────────────────────────────────────────────
签字栏（项目级一次）
─────────────────────────────────────────────
```

最终输出：封面/总览/章节出现 1 次；`{{表格:锚杆}}` 处展开成 N 张逐根表（表2.4-1…N，序号自增、曲线图嵌好）；签字栏出现 1 次。没勾的 `{{表格:xxx}}` 占位符不会出现在最终 docx 里。
